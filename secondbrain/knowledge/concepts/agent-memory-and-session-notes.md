# Память агента: фиксация работы в SecondBrain

## Политика (обязательная)

Всё значимое, что делается в этом репозитории, агент **должен отражать в SecondBrain**, чтобы память жила в файлах, а не только в чате. Цель — **не сканировать каждый раз весь проект**: сначала `knowledge/index.md` и релевантные страницы, затем точечное чтение кода.

## Что фиксировать

- Решения по архитектуре, стеку, границам модулей.
- Где что лежит (пути к ключевым файлам, entrypoints, конфиги).
- Контракты API, схемы данных, инварианты.
- Исправленные баги: симптом, причина, где починено (кратко).
- Договорённости с пользователем и операционные правила (как этот документ).

Не раздувать вики мелочами: форматирование, переименования без смысла, косметика — без отдельных страниц (см. `wiki-ingest-completion-gate` в правилах Cursor).

## Куда писать

1. **Новая устойчивая тема** — страница в `knowledge/concepts/` (или `connections/`, если связка нескольких тем).
2. **Обновление** существующей страницы, если тема уже есть.
3. **`knowledge/index.md`** — добавить строку в таблицу, если появилась новая значимая статья.
4. **`knowledge/log.md`** — строка `ingest | …` на сессию/крупный шаг.
5. При необходимости — **`knowledge/project-context.md`** (коротко, «что важно прямо сейчас»).

## Порядок работы агента

1. Перед разбором задачи: открыть **`secondbrain/knowledge/index.md`**, при необходимости `project-context.md` и 1–2 концепта.
2. Делать изменения в коде/конфигах.
3. После значимого результата — обновить вики (шаги выше), не откладывая «на потом».

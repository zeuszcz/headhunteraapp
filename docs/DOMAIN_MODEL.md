# Доменная модель headhunteraapp (роли, видимость, статусы)

## Роли

| Роль | Кто | Спрос / предложение |
|------|-----|---------------------|
| `company` | Заказчик | Публикует объекты (`work_objects`), просматривает **каталог исполнителей** (работники и бригады), отклики, чаты, аналитику по своим объектам. |
| `worker` | Исполнитель (человек) | Заполняет профиль, видит **ленту объектов** (вакансии/задачи) компаний, откликается, чат, отзывы. |
| `brigade` | Исполнитель (бригада) | То же, что у работника, с точки зрения сценариев «рынка». |

## Видимость (кто что видит)

- **Лента объектов** (`GET /api/v1/objects`): по умолчанию только объекты со статусами **опубликованные для рынка** (`open`, `in_progress`). Черновики (`draft`) в ленту не попадают.
- **Карточка объекта** (`GET /api/v1/objects/{id}`): черновик доступен **только владельцу-компании**; остальным при `draft` — 404.
- **Каталог исполнителей** (`GET /api/v1/profiles/workers`, `/brigades`, объединённо `/talent`): доступен **только роли company** (поиск подрядчиков). Исполнители не «паспортят» весь рынок компаний через этот API — у них отдельные сценарии (лента объектов, профиль компании по `user_id` при необходимости).
- **Публичный профиль** по `user_id` (`GET .../workers/{id}`, `.../brigades/{id}`, `.../companies/{id}`): карточка исполнителя/компании для просмотра после перехода из списка или отклика.

## Статусы объекта (`work_objects.status`)

| Статус | Смысл |
|--------|--------|
| `draft` | Черновик, виден только компании-владельцу. |
| `open` | Опубликован, в ленте у исполнителей. |
| `in_progress` | В работе (исполнение начато/договорённости достигнуты). |
| `closed` | Закрыт (набор/работы завершены). |
| `cancelled` | Отменён. |

Переходы задаёт бизнес-логика и UI компании (публикация из черновика, закрытие и т.д.).

## Основные потоки

1. **Компания**: создаёт объект (черновик или сразу `open`) → просматривает исполнителей в каталоге → открывает чат по объекту с выбранным `peer_user_id` → обрабатывает отклики.
2. **Работник/бригада**: настраивает профиль → лента объектов → отклик → переговоры в чате → статус отклика (принят/отклонён).

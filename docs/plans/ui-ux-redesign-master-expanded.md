# Мастер-план UI/UX: полная спецификация улучшений продукта

Документ задаёт **как удобнее пользоваться** приложением headhunteraapp: расположение, размеры, поведение, **новые страницы и функции**, идеи на будущее. Это живой ориентир для дизайна и разработки; реализация может идти **фазами**.

**Связь с кодом:** страницы в `frontend/src/pages/`, оболочка в `frontend/src/components/`, маршруты в `frontend/src/App.tsx`.

---

## 1. Для кого продукт и что значит «лучше для использования»

| Персона | Главная задача | Что «хорошо» на практике |
|---------|----------------|---------------------------|
| **Гость** | Понять сервис, зайти в ленту, зарегистрироваться | Мало шагов до цели, нет перегруза, доверие (понятный футер, нет «сломанных» экранов). |
| **Компания** | Опубликовать объект, найти исполнителя, вести переписку | Один явный primary на экране кабинета/формы; быстрый доступ к «Исполнителям» и чатам; видимость статусов. |
| **Работник / бригада** | Найти заказ, откликнуться, общаться | Лента читается быстро (сетка, фильтры не прячутся); отклик за 1–2 экрана; уведомления не теряются. |

**Принципы (приоритет):**

1. **Одна первичная кнопка** на экране там, где есть явное действие (создать, сохранить, применить фильтры).
2. **Одинаковая «скелетная» структура** внутренних страниц: крошки → заголовок + (опционально) действия справа → основная колонка → сайдбар на desktop.
3. **Предсказуемые размеры:** кликабельные зоны не уже **44×44 px** там, где ожидается палец (кнопки навбара, пагинация, крупные CTA).
4. **Обратная связь:** загрузка (skeleton), успех (toast или inline), ошибка (текст + что делать), пустые состояния с **одним** понятным следующим шагом.

### 2.2 Новые страницы / маршруты (идеи, по приоритету)

| Путь (предложение) | Для кого | Зачем улучшает продукт |
|--------------------|----------|-------------------------|
| `/settings` или `/account` | Все авторизованные | Email-уведомления вкл/выкл, смена пароля, язык (будущее), выход — **центр управления аккаунтом**, разгружает шапку. |
| `/objects/:id/edit` | Компания-владелец | Редактирование объекта **без дублирования** логики с `NewObject` (общая форма или общие поля). Сейчас правка только через API/неочевидно в UI. |
| `/help` или `/faq` | Все | Короткие ответы: как откликаться, что такое статусы, как работает чат — снижает нагрузку на поддержку. |
| `/u/company/:userId` (публичный просмотр) | Все | Карточка компании по `GET /api/v1/profiles/companies/{id}` — доверие, переход с объекта «узнать о заказчике». |
| `/404` | Все | Кастомная страница «страница не найдена» + ссылки на ленту и главную. |
| `/onboarding` (опционально, один раз после регистрации) | Новый пользователь | 3 шага: заполнить профиль → открыть ленту / создать объект — **снижает брошенность**. |

**Важно:** новые маршруты добавляют объём разработки; разумно ввести **фазу 1** без них, а **фазу 2** с `/settings` и `/objects/:id/edit`.

## 3. Численные нормы интерфейса (чтобы «не расползалось»)

### 3.1 Сетка и контейнеры

| Параметр | Рекомендация | Зачем |
|----------|--------------|--------|
| Макс. ширина контента | `--max-content`: 1120–1200px (уже ~1180) | Читаемость строки на десктопе. |
| Горизонтальные отступы `app-main` | 16px (моб), 20–24px (tablet+), единые токены `--space-*` | Визуальное выравнивание по краям. |
| Сайдбар `PageLayout` | ~280–320px при ≥1024px | Не съедает основной контент. |

### 3.2 Типографика (шкала)

| Уровень | Размер (ориентир) | Назначение |
|---------|-------------------|------------|
| H1 страницы | 1.5–1.75rem, font-weight 700 | Один на экран в `PageLayout`. |
| H2 секции | 1.1–1.25rem, font-weight 600 | «Фильтры», «Отклики», блоки формы. |
| Body | 0.95–1rem | Основной текст. |
| Подпись / мета | 0.8–0.88rem, цвет `--color-muted` | Крошки, подписи к полям, второй ряд в карточке. |

### 3.3 Кнопки (размеры и роли)

| Вариант | Min-height | Padding (ориентир) | Когда использовать |
|---------|------------|---------------------|---------------------|
| **Primary** | 44px (2.75rem) | 0.68rem 1.15rem | Одно главное действие: «Сохранить», «Применить фильтры», «Отправить отклик». |
| **Secondary** | то же | то же | Второе по важности: «Написать компании», «В избранное» на карточке (можно см. `btn--sm`). |
| **Ghost** | то же | то же | «Избранное», «Аналитика» в кабинете — не конкурируют с primary. |
| **Danger** | то же | — | «Отклонить», удаление подтверждённое модалкой. |
| **sm** | ~36px (2.25rem) | меньше padding | Внутри карточек и плотных списков. |

**Правило:** на одном экране не более **одной** primary, если нет двух независимых потоков (например форма + отдельный блок).

### 3.4 Поля формы

| Элемент | Норма |
|---------|--------|
| Высота input | ≥40px, скругление как у `.btn`/`--radius-sm`. |
| Между полями | 12–16px (`gap` в `.form-stack`). |
| Label | Над полем, font-weight 500, мелкий отступ до input. |
| Ошибка | Под полем или сверху формы блоком, цвет `--color-danger`. |

### 3.5 Брейкпоинты (для сетки ленты и навбара)

| Ширина | Поведение |
|--------|-----------|
| &lt; 640px | Лента: 1 колонка; фильтры столбцом; навбар: перенос групп или гамбургер **(отдельная задача)**. |
| 640–1023px | Лента: 2 колонки при возможности. |
| ≥1024px | Лента: 2–3 колонки; `PageLayout` с сайдбаром. |

## 4. Глобальная оболочка (детально)

### 4.1 Шапка `AppNav`

**Расположение (desktop):** слева бренд → группы ссылок с подписью группы мелким капсом → справа мета + «Выйти».

**Улучшения:**

- **Визуальное разделение групп:** лёгкий фон-пилюля или вертикальный разделитель между группами (не перегружать).
- **Активный пункт:** фон + цвет текста (уже есть класс active) — проверить контраст на новой палитре.
- **Уведомления:** если есть непрочитанные — бейдж с числом (нужен API или локальный счётчик) — **новая функция**, фаза 2.
- **Гость:** порядок «Объекты | Вход | Регистрация»; «Регистрация» — единственная заливка primary в гостевой шапке.

### 4.2 Подвал `AppFooter`

**Идеи:** добавить ссылку «Помощь» → `/help`; вторичная строка с версией или «MVP» для прозрачности.

### 4.3 Состояние загрузки `Protected`

- Skeleton **на всю ширину** `app-main`, не узкая карточка 520px — меньше «прыжка» при появлении контента.
- Опционально: скелет имитирует структуру целевой страницы (заголовок + 3 строки).

## 5. Дизайн-система: компоненты и модалки

### 5.1 Модалка `Modal`

| Зона | UX-деталь |
|------|-----------|
| Открытие | Фокус на панели; блокировка скролла body. |
| Закрытие | Крестик, клик по overlay, Escape. |
| Ширина | max-width 480–560px на десктопе; на мобильных — почти на всю ширину с отступами 16px. |
| Футер | Primary справа (подтверждение), Secondary слева (отмена) — привычный порядок для русскоязычных пользователей можно оставить «Отмена | Действие». |

**Улучшение:** focus trap (Tab не уходит за пределы модалки).

### 5.2 Toast / уведомления об успехе

**Сейчас:** частично inline «Сохранено», «Отклик отправлен».

**Идея:** единый компонент **Toast** (правый нижний угол, 3s, стек нескольких) — не перегружает форму, видно при скролле. Фаза 2.

### 5.3 Пагинация (новый компонент)

| Элемент | Поведение |
|---------|-----------|
| Показ | «Страница 3 из 12» или «Показано 25–36 из 140». |
| Кнопки | Prev / Next + при необходимости номера страниц (макс. 5–7 видимых). |
| Размер** | Высота кликабельной зоны ≥44px. |
| URL | `?page=3` синхронизировать с состоянием — можно поделиться ссылкой. |

## 6. Постранично: расположение, кнопки, состояния, идеи

Ниже для **каждой** существующей страницы: зоны сверху вниз, **каждая кнопка/ссылка**, пустые и ошибочные состояния, **идеи улучшений**.

### 6.1 `/` — Landing

| Зона | Содержимое | Элементы |
|------|------------|----------|
| Hero | Заголовок, текст | **Смотреть объекты** (primary), **Создать аккаунт** (ghost/outline) |
| (Если залогинен) Карта разделов | Сетка карточек | Каждая — вся карточка ссылка |
| Как работает | 3 шага | Без кнопок |
| Доверие | 2 блока текста | Без кнопок |

**Улучшения:** hero на полную ширость с мягким градиентом **в пределах новой палитры**; CTA дублировать внизу страницы для длинного скролла; **якорь** `#how-it-works` для ссылки из футера.

---

### 6.2 `/feed` — Лента

**Желаемый порядок блоков:**

1. Крошки  
2. Заголовок + короткий lead  
3. (Опционально) одна строка: **сортировка** «Сначала новые / по региону» — **новая функция**, если API поддержит `order=`  
4. Карточка **Фильтры:** поля в сетке 1 col (mobile) / 2–3 col (desktop), кнопка **Применить фильтры** (primary) справа или на всю ширину на мобиле  
5. Строка **«Найдено: N объектов»** + при пагинации «Страница k из m»  
6. **Сетка карточек** `FeedObjectCard` (равные ряды, фиксированное превью)  
7. **Пагинация** внизу  

**Кнопки/элементы:** Применить фильтры; в сайдбаре HelpHint «?» (модалка); ссылки внутри карточки — заголовок на объект.

**Пусто:** `EmptyState` — primary «На главную», текст объясняет сброс фильтров.

**Ошибка:** блок ошибки над фильтрами + **Повторить** (secondary).

**Идеи:** сохранённый поиск «Сохранить фильтр» (localStorage или API) — фаза 3; экспорт списка в CSV для компаний — низкий приоритет.

---

### 6.3 `/objects/:id` — Объект

| Зона | Элементы |
|------|----------|
| Шапка | Крошки, заголовок, подзаголовок с HelpHint статуса → модалка |
| Медиа | Обложка full width в колонке main |
| Компания | Карточка: имя + город; **идея:** ссылка «Профиль компании» на `/u/company/:id` когда появится страница |
| Факты | Список ul компактно |
| Тексты | Описание, оплата, навыки, контакт |
| Исполнитель | **Написать компании** (secondary) |
| Форма отклика | Поля + **Отправить отклик** (primary) |
| Отклики (компания) | Карточки; **Принять** / **Отклонить** / **Чат** |
| Модалка | Только при Отклонить |
| Отзыв | Оценка + текст + **Отправить отзыв** |
| Низ | **← К ленте** |

**Идеи:** sticky-бар на мобиле с одной кнопкой «Откликнуться»; для владельца — **Редактировать объект** → `/objects/:id/edit`.

### 6.5 `/dashboard`

**Компания:**

| Зона | Элементы |
|------|----------|
| Заголовок | actions: **Новый объект** (primary) |
| Вторичный ряд | Исполнители, Избранное, Аналитика (secondary/ghost) |
| Список | Карточки объектов |

**Пусто:** EmptyState — **Создать объект** + **Лента**.

**Исполнитель:** один primary **К объектам**; текст со ссылками на ленту и профиль.

**Идея:** виджет «У вас N непрочитанных уведомлений» ссылкой на `/notifications`.

Секции уже разбиты — **визуально:** нумерация шагов 1–2–3 в боковой колонке (сайдбар) как мини-оглавление; на мобиле — аккордеон свернуть «Требования» и «Сроки».

## 8. Дорожная карта внедрения (фазы)

**Фаза 1 — основа UX ленты и визуал**

- Токены палитры + пройти компоненты.  
- API: `offset`, `total`, контракт ответа.  
- Feed: сетка, пагинация, счётчик, URL.  
- Подправить модалку/кнопки под токены.

**Фаза 2 — завершение ключевых сценариев**

- Редактирование объекта.  
- Настройки аккаунта.  
- Toast для успеха.  
- Улучшение skeleton Protected.

**Фаза 3 — доверие и удержание**

- Help, публичный профиль компании, онбординг, сохранённые фильтры.

**Фаза 4 — полировка**

- Тёмная тема, расширенная аналитика UI, нижняя навигация на mobile.

---

## 9. Метрики успеха (ориентиры для продукта)

- Время от входа на `/feed` до клика по объекту (меньше — лучше после улучшения сетки).  
- Доля пользователей, заполнивших профиль >50% после онбординга (если введён).  
- Количество обращений «как пользоваться» (косвенно — после `/help`).
